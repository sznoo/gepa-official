You are judging semantic difference magnitude between HotpotQA retrieval states.

Given two R-state transitions, select the segment with the larger semantic change in retrieved/retrieval-context state.

Here, ΔR means the semantic difference between from_state and to_state:
- what evidence topic the retrieval state is centered on
- which entity/title/alias anchors are present or missing
- which bridge relation, comparison relation, or task relation is represented
- which evidence family or answer type the state is oriented toward
- which distractor entities, wrong senses, or noisy evidence families are introduced or removed
- how broad or narrow the retrieval state is as an evidence set

Important distinctions:
- Judge ΔR semantic magnitude, not prompt-feedback magnitude.
- Judge ΔR semantic magnitude, not BM25 query-token edit size.
- Judge ΔR semantic magnitude, not endpoint quality or answer correctness.
- Do not prefer a segment merely because the to_state is more verbose, more explicit, or more procedural.
- Do not prefer a segment merely because it mentions infoboxes, lead sentences, cast sections, table rows, or source styles.

- Retrieved titles are evidence of the retrieval state, but generated descriptive fields are also evidence. Read both as noisy descriptions of the same underlying R-state.
- If two transitions differ only by extraction specificity after the same semantic evidence target is already fixed, treat that as a small ΔR.
- If one transition changes the central evidence target, entity sense, bridge relation, or distractor family, treat that as a large ΔR.
